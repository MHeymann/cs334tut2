This course will be offered during the 2nd semester of 2017 by Brink van der Merwe. It will explain functional programming concepts by using Haskell.

Why should you take this course

Functional programming is a style of programming in which the primary method of computation is the application of functions to arguments. It offers a compact notation for writing programs, powerful abstraction methods for structuring programs, and a simple mathematical basis that supports reasoning about programs. Functional languages represent the leading edge of programming language design, and the primary setting in which new programming concepts are introduced and studied.

Lecture 7: Test

  • Define a tree type that has only one constructor. Instead of the Empty constructor, use the Maybe type to refer to a node's children.

  • Make your tree type an instance of the Show, Eq and Functor type classes. Provide examples to show that your fmap function works as expected.

  • Write a treeInsert function that will insert values into a tree in such a way that a binary search tree is obtained. Test your function treeInsert on a few small examples using foldr and treeInsert to build a binary search tree from a list of values.

  • Modify the treeFold function at cis194 to work on your tree data type. Test the treeFold function by implementing the flatten function using treeFold and testing flatten on a few small examples.

  • No Homework assigned this week, outside of the preparation listed for this week.

Lecture 8

  • Chapter 10: Learn You a Haskell for Great Good!

  • No extra reading this week

  • Homework: You should use the algorithmic idea from the London to Heathrow shortest path implementation in Chapter 10 of Learn you a Haskell for Great Good to implement the Viterbi algorithm as described at Wikipedia. Test you code on the ['Healthy', 'Healthy', 'Fever'] example at the bottom of the Wikipedia page. Note that your code should allow for arbitrary many hidden states and observations - i.e. not only 2 hidden states as in the Wikipedia example - i.e. you could have more than 2 parallel paths that you can cross over between if you think in terms of the London to Heathrow example. Find at least one more test case for your code having more than 2 hidden states. You should assume that as input you have the following as described on the Wikipedia page:

    • observation space

    • state space

    • a sequence of observations

    • transition matrix

    • emission matrix

    • an array of initial probabilities
